**为什么需要公钥和私钥（VAPID）**

核心问题是：FCM 怎么知道这条推送消息是你的服务器发的，而不是别人伪造的？

没有 VAPID 的话，任何人拿到用户的 `endpoint` 都能向他推送消息，这是安全漏洞。

VAPID 用**非对称加密**解决这个问题：

```
你生成一对密钥：
  私钥 — 只有你的服务器知道，绝不泄露
  公钥 — 可以公开，给浏览器和 FCM 知道
```

订阅时，浏览器把公钥告诉 FCM：

```
浏览器 → FCM：「我接受来自这个公钥对应服务器的推送」
```

推送时，你的服务器用私钥对消息签名：

```
服务器 → FCM：「这是我的签名，证明是我发的」
FCM 用公钥验签 → 验证通过 → 转发给浏览器
FCM 用公钥验签 → 验证失败 → 拒绝，保护用户
```

{
"crons": [{
"path": "/api/push/send-reminders",
"schedule": "0 9 * * *"
}]
}

```

`0 9 * * *` 是 cron 表达式，意思是「每天9点0分」。格式是：
```

分 时 日 月 周
0 9 \* \* _ → 每天9:00
0 _/2 \* \* _ → 每2小时
0 9 _ \* 1 → 每周一9:00

